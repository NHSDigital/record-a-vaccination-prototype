Accessibility statementt

This accessibility statement applies to the Record a vaccination service (www.ravs.england.nhs.uk)

This service is run by NHS England.

AbilityNet has advice on making your device easier to use if you have a disability.

How accessible this service is

We know parts of this service are not fully accessible. For example:

  • some content is not displayed correctly when you zoom in
  • some interactive elements are not announced to screen readers
  • some interactive elements are not accessible to keyboard users
  • some error messages are not optimised for screen readers and keyboard users
  • some tables have missing headers
  • some focus states are missing

Enforcement procedure

We want to improve the accessibility of the Record a vaccination service. If you find any problems not listed on this page or think we’re not meeting accessibility requirements, email ravs.support@england.nhs.uk.

Give feedback and report accessibility problems

The Equality and Human Rights Commission (EHRC) is responsible for enforcing the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018 (the ‘accessibility regulations’). If you’re not happy with how we respond to your complaint, contact the Equality Advisory and Support Service (EASS).

Technical information about this website’s accessibility

NHS England is committed to making this service accessible, in accordance with the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018.

Compliancde status

This service is partially compliant with the Web Content Accessibility Guidelines version 2.2 AA standard, due to the non-compliances listed below..

Non-accessible content

The content listed below is non-accessible for the reasons provided.

Various pages

  • Links from error summaries do not always go to the relevant input fields and the summary is not always properly focused to. (3.3.1 Error Identification and 3.3.3 Error Suggestion)
  • Some error messages appear when leaving required input fields. This is not announced by screen readers. (3.3.1 Error Identification and 3.3.3 Error Suggestion)
  • System JavaScript pop-ups are used in some places to signify an error. They can interrupt user flow, do not always work with assistive technologies and can affect focus management. (3.3.1 Error Identification)
  • Some tables have missing column headings. (1.3.1 Info and Relationships)
  • Some items are not highlighted by the keyboard focus indicator when accessed via keyboard navigation. (2.4.7 Focus Visible)
  • Some content is not displayed correctly when using your browser to zoom in. (1.4.10 Reflow)

Record a vaccination section

  • On the Find a patient page, Aria labels are used when not needed. (4.1.2 Name, Role, Value)
  • On the Find a patient page, when clicking on search, results appear at the bottom of the page. This is not announced to screen readers and focus moves to the top of the page. (1.3.2 Meaningful Sequence)
  • On the Patient details page, there are empty fieldsets in the tables which may cause confusion. (1.3.1 Info and Relationships) 
  • On the Patient details page, the 'Continue' button appears before the vaccination history. Screen reader users many never get to this content. (1.3.2 Meaningful sequence)
  • On the Assess and Vaccinate pages, the date picker does not work well with keyboard navigation and screen readers. (1.3.1 Info and Relationships) 
  • On the Vaccinate page, disabled fields are used for dose and expiry date which may cause confusion for some users. (1.3.3 Sensory Characteristics)
  • On the Check and confirm page, all the content is contained within a fieldset. This can create extra confusion for screen reader users. (1.3.1 Info and Relationships)
  • On the Check and confirm page, there are no focus states on the 'Change' links. (2.4.7 Focus Visible)
  • In the 'Record saved' banner, the link to 'View record' is not marked up correctly as a link so it is just read out as text to screen readers. (1.3.1 Info and Relationships and 1.3.5 Identity Input Purpose)

Vaccines section

  • There are multiple links named 'View product' and 'Add batch' in a table, with no visually hidden text. (1.3.1 Info and Relationships)
  • On the Choose site page, the autocomplete dropdown is not announced to screen readers. (3.3.2 Labels or Instructions, 4.1.2 Name, Role, Value and 4.1.3 Status Messages)
  • On the Choose vaccine page, the secondary options are not conditionally revealed. (1.3.1 Info and Relationships)
  • On the Vaccine batches page, the modal dialogue to reactivate a batch cannot be closed with the escape key. (2.2.2 Pause, Stop, Hide)
  • On the Vaccine batches page, the definition list to describe a vaccine is incorrectly using labels for the key value. (1.3.1 Info and Relationships)
  • When you add a vaccine, there is no confirmation message. This might cause confusion. (4.1.2 Name, Role, Value)

Manage users section

  • In Manage users, when you add a user, there is no confirmation message. This might cause confusion. (4.1.2 Name, Role, Value)

Header and banner

  • When clicking on the user's email address in the site header, it's not clear where the link goes and focus remains on the link. (2.4.4 Link Purpose and 2.4.3 Focus Order)
  • When selecting either option from the cookies banner, focus does not go back to the top of the page. (1.3.2 Meaningful Sequence)

What we’re doing to improve accessibility

We are working to fix all content that fails to meet the WCAG 2.2 AA standard.

We are planning significant changes to the service in the next 6 to 12 months.

Following these changes, we will carry out another assessment of the service and update this statement.

Preparation of this accessibility statement

This statement was prepared on [date when it was first published].

The Record a vaccination service (www.ravs.england.nhs.uk) was tested on 4 October 2024 against the WCAG 2.2 AA standard by the NHS Accessibility Lab team. We also used findings from the Record a vaccination team's own testing when preparing this statement.
